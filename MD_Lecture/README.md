## Введение / Философия

Markdown задуман как формат, который:

- максимально простой для чтения и редактирования как чистый текст. 
- почти не выглядит как «разметка» — то есть документ в Markdown должен быть читаем сам по себе. 
- поддерживает лишь небольшой поднабор HTML-тегов, основная задача — писать прозу, а не заменять HTML. 
- допускает вставку HTML там, где нужно (блоки HTML, span-элементы) — Markdown не пытается быть заменой всем HTML. 

Таким образом: Markdown — это скорее «формат для письма», чем полноценный язык разметки с полным покрытием HTML.

## Заголовки

Markdown поддерживает два стиля: Setext и ATX. 

- Setext-стиль:
```md
Это заголовок H1
=================

Это заголовок H2
-----------------
```

- ATX-стиль: используем # в начале строки (от 1 до 6 штук) для уровней H1–H6.
```md
# Заголовок H1
## Заголовок H2
###### Заголовок H6
```

## Цитаты (Blockquotes)

> Пример цитаты
> 
> - Пример
> - Цитата

- Используется символ > в начале строки, имитируя стиль цитирования в письмах. 
- Можно выставлять > только перед первой строкой абзаца (если абзац hard-wrapped) или перед каждой строкой. 
- Поддерживается вложенность: для вложенной цитаты ставим >>, или > > и т. д. 
- Внутри цитаты можно использовать другие элементы Markdown: заголовки, списки, кодовые блоки и т.д

## Списки

* Элемент 1
+ Элемент 2
  + Вложенный пункт
- Элемент 3
1. Первый
2. Второй
    1. Вложенный пункт
3. Третий

Нумерованные (упорядоченные) и маркированные (неупорядоченные).

Неупорядоченные: маркеры *, +, - — любые из них можно использовать. Пример:

```md
* Элемент 1
+ Элемент 2
  + Подпункт
- Элемент 3
```

Упорядоченные: цифра + точка + пробел:

```md
1. Первый
2. Второй
    1. Вложенный пункт
3. Третий
```

## Горизонтальная линия (Horizontal Rule)

Горизонтальная черта создаётся если на строке, отдельно от текста, стоит три или более символов -, * или _, возможно с пробелами между ними. Примеры:

* * *
***
*****
- - -
---------------------------------------

```md
* * *
***
*****
- - -
---------------------------------------
```

## Ссылки

- Есть два стиля: inline (встроенные) и reference (ссылки-определения). 
- Inline-ссылка: [текст ссылки](http://example.com/)
```md
[текст ссылки](http://example.com/)
```
- Reference-ссылка: 
  - сначала в тексте: 
  ```md
  [текст ссылки][идентификатор]
  ```
  - А потом где-то ещё (часто в конце документа) определить: 
  ```md
  [идентификатор]: http://example.com/ 
  ```
[текст ссылки][example-link]

[example-link]: http://example.com/ 

- Если используется пустой идентификатор [], тогда текст ссылки служит именем:
```md
[Google][]
[Google]: http://google.com/
```

[Google][]

[Google]: http://google.com/

## Выделение (Emphasis)

- Используйте * или _ для выделения. Один символ для курсива, два — для жирного.
  - *курсив*  
  - _курсив_  
  - **жирный**  
  - __жирный__
  ```md
  - *курсив*  
  - _курсив_  
  - **жирный**  
  - __жирный__
  ```
- Можно вставлять внутри слова: un*frigging*believable. `un*frigging*believable`
- Если символ (* или _) окружён пробелами, он трактуется как обычный символ, а не как маркер. 
- Если нужно вывести символ * или _, который воспринимается как маркер, можно экранировать его обратным слэшем: \*literal asterisks\* `\*literal asterisks\*`

## Код внутри строки (Code span)
Чтобы обозначить кусочек кода внутри параграфа, используем обратные кавычки (\`):

Use the `printf()` function.

```md
Use the `printf()` function.
```

## Изображения

- ![cat](./cat_1.jpeg)
- ![cat][cat2]

[cat2]: ./cat_2.jpeg

- Синтаксис как у ссылки, но начинается с !. 
  - Inline-изображение:
  ```md
  ![Alt текст](/path/to/img)
  ```
  - Reference-изображение:
  ```md
  ![Alt текст][id]
  [id]: /path/to/img
  ```

## Автоматические ссылки

<http://example.com/>

- Если вы помещаете URL или email-адрес в угловые скобки, Markdown преобразует их в HTML ссылку:
```md
<http://example.com/>
```

## Экранирование обратным слэшем
Если нужно вывести символы, которые обычно считаются маркерами разметки, можно поставить перед ними `\`, чтобы они воспринимались буквально. 

Список символов, которые можно экранировать: \\, \`, \*, \_, \{\}, \[\], \(\), \#, \+, \-, \., \!.
